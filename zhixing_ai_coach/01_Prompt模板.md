> 用途：每次用户付费后，将此Prompt（含用户信息）输入 Claude/GPT，生成定制训练方案
> 使用方式：复制下方完整Prompt，在 {用户信息} 处填入实际数据，发送给AI后直接产出方案

**V3强化要求：**
- 如果用户是私教转自己练 → 所有动作必须是ta之前跟教练做过的。标注"你练过的"或"你上课做过"。

## 方案生成后的自检清单

**在输出方案前，逐一检查以下12个维度（V3从9项扩展到12项）：**

- [ ] 1. 目标匹配度：方案的主要训练方式是否直接服务于用户的核心目标？
- [ ] 2. 器械可用性：方案中每个动作所用器械是否都在用户的可用列表中？**特别注意哑铃重量上限**。
- [ ] 3. 动作安全性：是否基于用户的伤病史/疼痛排除了风险动作？（膝关节/下背/肩关节/踝关节/术后左右不均衡）
- [ ] 4. 渐进负荷：是否有明确的逐周递增计划？第4周是否设置为减载+测试？
- [ ] 5. 可执行性：总时长在用户时间预算内？**有备用方案（精简版/在家版）吗？**
- [ ] 6. 动作多样性：不同训练日之间是否有足够的动作变化？
- [ ] 7. 专业价值：每个动作要点是否包含呼吸+纠偏+发力感三要素？发力感是否用了比喻？
- [ ] 8. 生活适配度：是否考虑了应酬/用餐/睡眠/工作压力/经期等生活因素？
- [ ] 9. 备用方案：是否提供了多级备用方案？（时间紧张版+完全不想出门版+伤病降级版）
- [ ] 10. 心理适配：【V3新增】方案的复杂度和语气是否匹配用户当前的心理状态？（高压工作=先肯定+做减法。停训=先给成绩单。同行=平等+可迁移）
- [ ] 11. 教练-会员关系处理：【V3新增】如果是私教转自己练→动作是否是ta做过的？如果不续课→是否表达了真诚关怀？如果用户有"丢脸感"→是否在方案中处理了？
- [ ] 12. 完成标准：【V3新增】是否给用户定义了清晰的"完成标准"？（"去了就完成""做了就完成"——不追求完美）

**如果以上有任何一项没有通过，返回修改后再输出。**

## 使用说明

### 使用流程
1. 用户付费后，发送腾讯问卷链接（见《02_问卷字段设计.md》）
2. 用户填写完成 → 你收到数据
3. 按《00_技能-计划制定.md》决策树识别会员类型
4. 将用户数据填入上方Prompt的 { } 占位符中
5. 将完整Prompt发送给Claude/GPT
6. AI输出方案后，**用自己的专业判断过一遍**——以你的判断为准
7. 将方案整理到交付模板（见《03_方案交付模板.md》），生成Excel+说明书
8. 发给用户 + 24h跟进
9. 将新案例的策略创新回写到《00_技能-计划制定.md》
