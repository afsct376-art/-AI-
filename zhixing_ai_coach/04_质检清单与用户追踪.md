# 知行AI教练 · 方案交付前质检Checklist V2

> 用途：每份方案交付给用户之前，你（崔知行本人）亲自过一遍此清单
> 原则：前50单每单必检，50单后随机抽查20%

---

## 交付前11项检查

**每份方案发出前，逐项打勾：**

- [ ] **□ 1. 方案与用户目标对齐**
  - 用户的主要目标在方案中有明确体现
  - 用户有多个目标时，方案是否同时覆盖？（如"减脂增肌+学会做计划"→方案是否包含了自主训练框架？）
  - 目标细节在渐进负荷和复盘节点中有呼应

- [ ] **□ 2. 器械匹配 + 上限检查** 【V2强化】
  - 方案中每个动作所用器械 → 用户可用的器械列表中有
  - **哑铃重量建议不得超过用户实际可用最大重量**（如哑铃最重20kg→方案中所有哑铃动作重量建议≤20kg）
  - 固定器械动作 → 用户确认拥有该器械
  - **如果用户没有杠铃/深蹲架，方案中不应出现杠铃深蹲等动作**
  - 如果已有动作触及器械上限，是否提供了突破策略（tempo/1.5次法/缩短休息/增加次数/单侧变式）？

- [ ] **□ 3. 关节安全与专项保护** 【V2新增】
  - 基于用户的关节不适信息，排除了风险动作
  - 如果用户有关节不适（膝关节/肩关节/下背），方案中是否有**关节保护专项模块**？
  - 关节保护模块是否包含：🔴禁止 / 🟡谨慎 / 🟢安全替代 / 🟢康复辅助 / ⚠️停止信号？
  - 绝对没有在用户关节不适的情况下写该关节的高风险动作（如膝盖不适还写腿屈伸、下背不适还写传统硬拉）

- [ ] **□ 4. 时间合理 + 精简备用** 【V2强化】
  - 热身+力量+有氧+拉伸总时长在用户可用时间+5分钟内
  - **如果时间紧张，方案中是否有削减建议（如"如果时间紧去掉⑤⑥"）？**
  - 是否有精简版/备用方案，让用户在无法完成完整计划时有退路？

- [ ] **□ 5. 方案有"教练感"** 【V2强化】
  - 每个动作要点同时包含：☑ 呼吸节奏 ☑ 常见错误纠偏（至少1个关键纠偏点）☑ 发力感提示
  - **发力感使用生活化比喻而非解剖学术语**
  - 例：*"想象把杠铃推弯"* ✅ / *"哑铃飞鸟时想象抱一棵大树"* ✅
  - 而非：*"保持核心收紧，三角肌前束发力"* ❌
  - 而非：*"躺在平凳上，双手持哑铃向两侧打开，然后举起"* ❌

- [ ] **□ 6. 渐进负荷清楚 + 多种进阶路径** 【V2强化】
  - 用户知道第1周和第4周的区别
  - **如果器械有上限（如哑铃只到20kg），方案中是否包含重量外进阶策略？**
  - 用户知道什么情况下加重量、什么情况下减、什么情况下换动作
  - 进阶判断规则是否具体可执行（不是"感觉重了就减，感觉轻了就加"这种废话）

- [ ] **□ 7. 有氧安排合理** 【V2新增】
  - 有氧方式是否基于用户可选择的方式？
  - 是否有备用有氧方案（跑步机坏了怎么办）？
  - 有氧强度是否有理由说明（为什么不是高强度）？
  - 有氧时长+力量训练总时长是否在用户时间预算内？

- [ ] **□ 8. 营养建议场景化** 【V2新增】
  - 营养建议是否基于用户实际用餐场景（而非泛泛的"多吃蛋白质"）？
  - 是否针对用户的应酬频率给出了应对策略？
  - 是否声明了"非医学处方"？
  - 建议是否具体到用户能执行（如"米饭控在一拳大小"而非"碳水200g"）？

- [ ] **□ 9. 排版感专业**
  - 排版整洁、无错别字、格式一致
  - 如果是文字版，分隔清晰，无AI常见的"最终润色"废话
  - 表格是否对齐？emoji是否使用恰当？
  - 方案完成后，整体看起来"值这个价"

- [ ] **□ 10. 用户被"教"到了吗？** 【V2新增】
  - 如果用户表达了"想学自己做计划"→方案中是否有自主训练框架？
  - 方案读完后，用户是否理解了"为什么选这个训练方式"？
  - 方案读完后，用户是否知道"怎样的信号代表可以做怎样的调整"？

- [ ] **□ 11. 泡沫轴/弹力带建议** 【V2新增】
  - 如果用户没有泡沫轴/弹力带且方案中需要用到→是否有建议购置清单？
  - 如果用户有泡沫轴/弹力带→热身和冷身中是否明确写出了使用方法？

## 常见AI错误 & 你的纠偏 V2

| AI容易犯的错 | 你应该怎么改 | V2新增 |
|-------------|-------------|--------|
| 动作太理论化（"保持核心收紧"一句带过） | 补充具体的发力感和纠偏提示 | 发力感用生活化比喻 |
| 忽略伤病史（有腰痛还写硬拉） | 替换为替代动作（如臀推/俯身腿弯举） | 新增关节保护专项模块 |
| 每周计划一样（没有渐进负荷） | 手动加入逐周递增逻辑 | 增加重量外进阶策略 |
| 时间超预算（60min写了90min的内容） | 减少动作种类或组数 | 增加"时间紧时削减建议" |
| 器械不存在（推荐了健身房固定器械，但用户在家庭训练） | 全部替换为可用器械的动作 | **新增哑铃上限检查** |
| 营养建议太绝对（"每天吃200g蛋白"） | 改成"建议方向"，加免责声明 | 按用户实际用餐场景给建议 |
| **忽略关节不适的渐进恢复** 【V2新增】 | 对有关节不适的会员，方案必须有康复辅助动作和停止信号 | — |
| **给所有人同样的分化方式** 【V2新增】 | 每周3练→全身训练；每周4练→上下肢；每周5-6练→推拉腿 | — |
| **有氧建议"一刀切"** 【V2新增】 | 结合用户有氧偏好、时间预算、目标给出针对性建议 | — |
| **忽略应酬对训练的影响** 【V2新增】 | 有应酬的用户→必须有应酬日调整策略+精简备用方案 | — |
| **只给方案不给能力** 【V2新增】 | 想学做计划的用户→附上自主训练框架 | — |

## 需优化Prompt的信号（V2补充）

**当出现以下任何信号时，需要优化Prompt：**

| 信号 | 严重程度 | 可能的原因 |
|------|---------|-----------|
| 连续3个用户评分低于4分 | 🔴 高 | 方案质量/匹配度有问题 |
| 超过50%的方案需要你手动大幅修改 | 🟡 中 | Prompt表达不够精准或缺失关键分支 |
| 有用户说"跟Keep上的差不多" | 🔴 高 | 方案缺乏个性化——可能是用户信息采集不够详细，或Prompt没有充分利用已有信息 |
| **用户执行2周后反馈"练不下去"** 【V2新增】 | 🟡 中 | 方案可执行性不足——时间太长/太难/与生活冲突 |
| **多个用户的器械上限问题没被解决** 【V2新增】 | 🟡 中 | Prompt缺少器械上限突破策略的输出要求 |
| **有关节不适的用户反馈"练完更疼了"** 【V2新增】 | 🔴 高 | 安全筛查不充分或方案未排除风险动作——需要立即修正 |

## V2 相比 V1 的变化

| 维度 | V1 | V2 |
|------|----|----|
| 检查项数量 | 7项 | **11项** |
| 关节安全 | 笼统的"安全检查"一项 | 独立为第3项——关节保护专项，要求包含🔴🟡🟢⚠️结构 |
| 器械匹配 | 仅检查有无 | 增加哑铃上限检查、固定器械逐台核对、上限突破策略检查 |
| 渐进负荷 | 仅检查是否有逐周递增 | 增加"重量外进阶策略"检查、多种进阶路径 |
| 营养建议 | 无专项检查 | 新增第8项——营养建议是否场景化、是否有应酬策略 |
| 有氧安排 | 无专项检查 | 新增第7项——方式是否基于用户偏好、是否有备用方案 |
| 用户赋能 | 无专项检查 | 新增第10项——用户是否被"教"到了东西 |
| 泡沫轴/弹力带 | 无 | 新增第11项——建议购置+融入热身冷身 |
| AI常见错误 | 6条 | **11条**，新增5条来自实战案例 |
| 满意度追踪表 | 5列 | **7列**，增加"最有用的"和"修改了什么" |
| 优化信号 | 3个 | **6个**，新增执行不下去、器械上限、关节不适反馈3个信号 |
