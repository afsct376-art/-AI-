## 产品哲学

**不是卖训练计划，是解决"为什么还没动"。**

AI负责效率，教练负责专业。每一份方案都是AI和教练共同完成的——AI出框架，教练用经验注入灵魂。

**核心洞察**：用户说的"目标"和真正需要的"方案"往往不是一回事。先识别ta卡在哪一步，再决定方案形态。

## 一、会员分类决策引擎（可编程化）

### 输入维度（10维 → 输出类型）

```
D1: 训练经验（0=零基础 1=初学者/重来 2=中级 3=高级 4=同行教练 5=私教会员转自己练）
D2: 训练频率（每周天数）
D3: 器械条件（0=徒手 1=固定器械 2=哑铃≤20kg+固械 3=杠铃齐全 4=商业健身房完整）
D4: 伤病约束（0=无 1=膝关节 2=下背 3=肩关节 4=术后 5=痛风/踝关节 6=其他）
D5: 生活约束（应酬/夜班/出差/工作压力/女性专项）→ V4结构化：工作性质+班型+夜班频率+通勤
D6: 心理状态（0=极度渴望没行动 1=有基础遇瓶颈 2=停训信心消退 3=已有信任 4=同行 5=工作压力大+内疚感）→ V4从问卷Q19f直接映射
D7: 训练经历（私教/自己练/两者混合）→ V4新增：中断原因+最长周期+教练经历
D8: 特殊需求（备赛/技能学习/在家+健身房混合）
D9: 女性专项（V4新增：经期影响程度 0=不影响 1=前1-2天需休息 2=前3天需休息 3=经期全程影响）
D10: 动机模块（V4新增：最长坚持周期+期望见效时间+最低可接受替代）
```

输出类型：
  TYPE_A = 渐进入门型（王丽颖模式）——零基础 入不了门
  TYPE_B = 重建信心型（蜡笔小杨模式）——停训后信心消退
  TYPE_C = 进阶突破型（Joe何模式）——中级 器械/伤病瓶颈
  TYPE_D = 精细优化型（毛丹模式）——耐力运动员+术后
  TYPE_E = 同行教练型（吉/瑞克模式）——健身教练 给同行的方案
  TYPE_F = 老会员进阶型（重庆会员模式）——已有信任 直接进阶
  TYPE_G = 高压工作型（小冯模式）——工作压力大+内疚+训练中断 做减法
```

### 决策树伪代码

```python
def classify_member(profile):
    # Step 1: 训练经验+心理状态 → 方案复杂度
    if profile.psychology == 5:  # 工作压力+内疚 = 先肯定再做减法
        plan_type = TYPE_G  # 高压工作型
        core_strategy = "ABC三方案+成绩单前置+完成比完美重要"
    elif profile.experience == 0:
        plan_type = TYPE_A
    elif profile.is_restart and profile.psychology == 2:
        plan_type = TYPE_B
    elif profile.is_coach:
        plan_type = TYPE_E
    elif profile.experience >= 3:
        plan_type = TYPE_D
    elif profile.experience == 2:
        plan_type = TYPE_C
    else:
        plan_type = TYPE_F
    
    # Step 2: 工作类型+班型 → 时间分配策略  【V4新增】
    if profile.work_type == "shift" or profile.night_shift_days > 4:
        # 轮班/夜班→夜班专项策略+双路径
        night_shift_strategy = "双路径方案——夜班日补觉+白班日训练"
        frequency_model = "不固定周频——按班型动态调整"
    if profile.commute_time >= 60:  # 通勤≥60min
        time_budget_adjustment = -15  # 可用训练时间减15分钟
    
    # Step 3: 频率+约束 → 分化方式
    if plan_type == TYPE_G:  # 高压工作型
        split = "两分化（上肢日/下肢日轮换）"
        frequency_model = "ABC三方案（精力好=完整版 累=精简版 不想出门=在家版）"
    elif profile.frequency <= 2:
        split = "全身训练×2"
    elif profile.frequency == 3:
        if profile.is_coach:
            split = "DUP 3天 或 推/拉/腿+高翻专项"
        else:
            split = "三分化（推/拉/腿）或 全身×3"
    elif profile.coach_member_transition:  # 私教转自己练
        split = "两分化——先用做过的固定器械动作 减少认知负荷"
    else:
        split = "上下肢分化"
    
    # Step 4: 女性专项 → 经期调整 【V4新增】
    if profile.gender == "female" and profile.menstrual_impact > 0:
        enable_pms_module = True
        if profile.menstrual_impact <= 1:
            pms_strategy = "经期前2天降级为精简版，之后恢复正常"
        elif profile.menstrual_impact <= 2:
            pms_strategy = "经期前3天切换为在家版，后2天过渡"
        else:
            pms_strategy = "经期全程切换为精简方案，经期后恢复正常"
    
    # Step 5: 动机模块 → 预期管理 【V4新增】
    if profile.max_streak < 1:  # 从未坚持超过1个月
        expectation_strategy = "短期反馈密集——每周一次进度确认"
        plan_complexity = "极简——先养成习惯再优化内容"
    if profile.expect_result_time == "2周内":
        expectation_strategy = "可视化设计——每周拍照+围度对比"
    
    # Step 6: 高峰时段 → 动作替换策略 【V4新增】
    if "17-20点" in profile.peak_hours:  # 高峰期训练
        equipment_strategy = "提供高峰期备选动作——热门器械可能被占用"
    
    # Step 7: 训练经历 → 动作库和自主感
    if profile.trained_with_coach and profile.now_self_training:
        # 小冯模式——所有动作都是教练带过的
        action_strategy = "全部使用已知动作库——不引入新动作 减少认知负荷"
        safety_strategy = "明确告知'不是你的问题 是动作的问题'——消除丢脸感"
```

### 策略模块库（V3 -> V4扩展）

| 模块ID | 模块名 | 激活条件 | 内容要点 |
|--------|-------|---------|---------|
| MOD_ABC | ABC三方案 | psych=5（高压工作） | 方案C完整版30-50min / 方案B精简版15-20min / 方案A在家15min |
| MOD_SCORECARD | 成绩单封面 | psych=5且有历史数据 | 封面第一屏展示之前的体重/腰围/维度变化——先肯定再建议 |
| MOD_KNOWN_ACTIONS | 已知动作库 | 私教转自己练 | 全部使用教练带过的动作——不引入新动作 减少认知负荷 |
| MOD_NOBLAME | 消除丢脸感 | 有训练挫折记录 | "腰疼不是你的问题 是动作的问题" "小臂酸=没热身+重量大" |
| MOD_DEPARTURE | 离别关怀 | 不续课 | 最后一句："不管续不续 我都希望你练得好" |
| MOD_DECISION_TREE | 渐进决策树 | exp≥2或同行 | 各训练日加重/维持/退阶/换动作决策表 |
| MOD_MENTAL | 心理策略 | 所有类型(按psych定制) | 状态差降级+不想练的心理策略 |
| **MOD_PMS** | **经期周期调整** | **女性+经期影响>0** | **V4新增：经期前中后的训练降级+恢复方案** |
| **MOD_HOUR** | **高峰期替代策略** | **高峰时段训练** | **V4新增：热门器械被占用的动作替换预案** |
| **MOD_COMMUTE** | **通勤时间适配** | **通勤≥60min** | **V4新增：时间预算压缩+精简策略** |
| **MOD_NIGHTSHIFT** | **夜班专项** | **夜班频率≥4天/月** | **V4新增：双路径方案——夜班补觉vs白班训练** |
| **MOD_EXPECTATION** | **预期管理** | **期望见效时间≤1个月** | **V4新增：密集反馈节点+可视化设计** |
